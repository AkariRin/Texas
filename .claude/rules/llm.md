# LLM 集成规则（强制执行）

## 模型配置

- 生产环境模型通过 `LLMService`（`src/services/llm.py`）统一管理，**禁止**在业务代码中硬编码模型名称
- 每次 LLM 调用必须设置合理的 `timeout` 和 `max_tokens` 上限，禁止无限等待
- 流式输出场景使用 `llm_stream`（`src/services/llm_completion.py`）避免请求超时

## 上下文窗口管理

- 发送给 LLM 的历史消息必须有**截断策略**（token 预算或消息条数上限），不得无限增长
- System prompt 保持精简，避免重复性描述；变更 system prompt 时评估 token 影响
- 长上下文场景优先考虑**摘要压缩**（将旧消息摘要后替换原始消息）而非直接截断头部
- 禁止在每轮对话中重复发送大块静态上下文（如全量文档），使用 RAG 或工具调用替代

## 调用规范

- LLM 调用失败必须有明确的重试策略（指数退避）和最终的降级处理，不得静默失败
- 禁止在请求链路的关键路径上同步等待 LLM 响应超过合理阈值；耗时推理任务走 Celery 异步
- 用户可见的 LLM 输出必须做基础内容过滤，防止模型返回敏感或有害内容直接透传
